1. Яка різниця між структурною та атрибутивною директивою?
Структурна диреткива використовується для формуання чи переформування HTML DOM щлящом додавання чи видалення елементів.
Директиви атрибутів використовуються для зміни зовнішнього виду або поведінки елемента. Атрибутивні директиви не мають можливості змінювати структуру DOM.
2. Що таке trackByFn?
При використанні ngFor для зациклювання масиву в шаблонах, звертайтеся до функції trackBy, яка повертає унікальний ідентифікатор кожного елемента.
3. Яка різниця між ngIf та display: none?
Головне значення між angular ngIf directive & hidden or display:none є те що ngIf додасть або видалить елемент з DOM на основі умови або виразу, а display:none окаже або приховає елемент HTML.
4. Пояснити decimal pipe
Decimal Pipe — це один із вбудованих каналів у Angular, який використовується для форматування десяткових чисел відповідно до даних про десяткові цифри та інформації про місцевість.

3. Яка різниця між @ViewChild і @ContentChild()?



@ContentChild повертає директиву, що існуює всередині елемента , тоді як @ViewChild переглядає тільки елементи, які знаходяться безпосередньо в шаблоні представлення.

4. Для чого нам providedIn: ‘root’ ?



Значення root для параметра providedIn у декораторі Injectable дозволяє встановити кореневий рівень дій. Кореневий рівень (root) передбачає дію сервісу для всієї програми. Створюється та використовується один об'єкт сервісу для всіх частин програми.
Відповіді на питання

1. Яка форма краще підходить для великих проектів?



Для великих проектів краще підходить реактивна форма.

2. Опишіть компоненти життєвого циклу: OnInit, OnChanges, AfterViewInit



ngOnInit -викликається один раз після встановлення властивостей компонента, що беруть участь у прив'язці. Виконує ініціалізацію компонента
ngOnChanges -викликається до методу ngOnInit() при початковій установці властивостей, які пов'язані механізмом прив'язки, а також при будь-якій їхній переустановці або зміні їх значень. Даний метод як параметр приймає об'єкт класу SimpleChanges, який містить попередні та поточні значення властивості.
AfterViewInit -викликається фреймворком Angular після ініціалізації уявлення компонента, а також уявлень дочірніх компонентів. Викликається лише один раз відразу після першого виклику методу ngAfterContentChecked()

3. Опишіть вбудовану перевірку в Angular



Перед тим, як надіслати форму, нам треба переконатися, що ця форма містить коректні значення. Для перевірки використовується механізм валідації.
Для перевірки коректності нам достатньо подивитися на властивість valid.
Однак не завжди може бажано відразу відображати помилку під час завантаження сторінки.
У цьому випадку ми дивимося на валідність, якщо поле введення вже отримувало фокус.
